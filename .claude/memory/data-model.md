# Data Model (Firestore)

## Collections — Field-Level Detail

### apiKeys (doc ID: `ts_<tier>_<uuid32hex>`)

| Field | Type | Description |
|---|---|---|
| `tier` | string | `free`, `pro`, or `business` |
| `name` | string | Display name (default: `"Unnamed"`) |
| `email` | string/null | Optional, set on signup |
| `active` | boolean | `false` = soft-revoked |
| `created` | Timestamp | `FieldValue.serverTimestamp()` |

### usage (doc ID: same as apiKeys)

| Field | Type | Description |
|---|---|---|
| `total` | number | All-time request count |
| `currentMonth` | string | `"YYYY-MM"` (zero-padded) |
| `currentMonthCount` | number | Requests this month |
| `lastUsed` | Timestamp | Last request time |

## Usage Tracking Mechanics

`trackAndEnforce()` in `usage.mjs`:
1. First call for a key → `set()` with count 1
2. Month rollover (stored month != current) → reset `currentMonthCount` to 1
3. At/over limit → reject **before** incrementing (no off-by-one)
4. Normal → `FieldValue.increment(1)` for atomic concurrent writes

## Key Format

`ts_<tier>_<32-hex-chars>` — generated by `uuid.v4()` with dashes removed. Regex: `/^ts_(free|pro|business)_[a-f0-9]{32}$/`

## Firestore Client

Lazy singleton via `getDb()` in `firestore.mjs`. Uses Application Default Credentials (ADC) — auto-detected on Cloud Run, requires `gcloud auth application-default login` locally.
