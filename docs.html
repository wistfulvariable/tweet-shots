Getting Started

tweet-shots converts tweet URLs or IDs into pixel-perfect PNG/SVG screenshots — no browser automation needed. Choose from multiple themes, gradient backgrounds, dimension presets, and styling options to create beautiful social media images programmatically.

1

Get a free API key

Sign up at /billing/signup or use the API directly:

curl -X POST https://tweet-shots-api-1084185199991.us-central1.run.app/billing/signup \
  -H "Content-Type: application/json" \
  -d '{"email": "you@example.com"}'
2

Take your first screenshot

Pass any tweet URL or ID as a path parameter:

curl "https://tweet-shots-api-1084185199991.us-central1.run.app/screenshot/1617979122625712128" \
  -H "X-API-KEY: your-api-key" \
  -o tweet.png
3

View the result

Open tweet.png to see your pixel-perfect screenshot. Customize with query parameters like ?theme=light&gradient=ocean.

Base URL: https://tweet-shots-api-1084185199991.us-central1.run.app

Authentication

All screenshot and tweet data endpoints require an API key. There are two ways to authenticate:

Option 1: HTTP Header (recommended)

curl "https://tweet-shots-api-1084185199991.us-central1.run.app/screenshot/1617979122625712128" \
  -H "X-API-KEY: ts_free_your-key-here"

Option 2: Query Parameter

curl "https://tweet-shots-api-1084185199991.us-central1.run.app/screenshot/1617979122625712128?apiKey=ts_free_your-key-here"

Security tip: Always use the header method when possible. Never expose your API key in client-side JavaScript or public repositories. The query parameter method is provided for convenience in simple scripts and testing.

Screenshot (GET)

GET /screenshot/:tweetIdOrUrl
Auth: Required Credits: 1 per request Response: Binary image

Generate a screenshot of a tweet and return the image directly. The URL parameter accepts either a numeric tweet ID or a full tweet URL (URL-encoded).

URL Parameter

NameDescription
tweetIdOrUrl A tweet ID (e.g., 1617979122625712128) or a full URL-encoded tweet URL (e.g., https%3A%2F%2Fx.com%2Felonmusk%2Fstatus%2F1617979122625712128). Both twitter.com and x.com URLs are supported.

Query Parameters

All query parameters are optional. See the Parameters Reference for the full list with types and allowed values.

theme, dimension, format, scale, outputWidth, gradient, bgColor, textColor, linkColor, hideMetrics, hideMedia, hideDate, hideVerified, hideShadow, hideQuoteTweet, showUrl, padding, radius, fontFamily, fontUrl, fontBoldUrl, logo, logoPosition, logoSize, frame, gradientFrom, gradientTo, gradientAngle, thread

Response Headers

Content-Type, X-Tweet-ID, X-Tweet-Author, X-Render-Time-Ms, X-Credits-Limit, X-Credits-Remaining

Example

curl "https://tweet-shots-api-1084185199991.us-central1.run.app/screenshot/1617979122625712128?theme=dark&gradient=ocean&dimension=instagramFeed" \
  -H "X-API-KEY: ts_free_your-key-here" \
  -o tweet.png

Screenshot (POST)

POST /screenshot
Auth: Required Credits: 1 per request Content-Type: application/json

Generate a screenshot with a JSON request body. Supports three response modes: binary image, base64-encoded JSON, or a hosted URL. Use proper JSON types (booleans, numbers) instead of strings.

Request Body

FieldTypeRequiredDescription
tweetIdstring*Tweet ID. Required if tweetUrl is not provided.
tweetUrlstring*Tweet URL. Required if tweetId is not provided.
responsestringNoResponse format: "image" (default), "base64", or "url".
...plus all render options from the Parameters Reference. Use proper types: booleans (true/false), numbers, not strings.

Aliases: You can use backgroundGradient in place of gradient, backgroundColor in place of bgColor, and borderRadius in place of radius.

Response Modes

response: "image" (default)

Returns the raw binary image with appropriate Content-Type header (image/png or image/svg+xml).

response: "base64"

Response
{
  "success": true,
  "tweetId": "1617979122625712128",
  "author": "elonmusk",
  "format": "png",
  "data": "iVBORw0KGgoAAAANSUhEUg..."
}

response: "url"

The image is uploaded to Cloud Storage and a public URL is returned.

Response
{
  "success": true,
  "tweetId": "1617979122625712128",
  "author": "elonmusk",
  "format": "png",
  "url": "https://storage.googleapis.com/tweet-shots-screenshots/screenshots/1617979122625712128-1709312000000.png"
}

Example

curl -X POST https://tweet-shots-api-1084185199991.us-central1.run.app/screenshot \
  -H "X-API-KEY: ts_free_your-key-here" \
  -H "Content-Type: application/json" \
  -d '{
    "tweetId": "1617979122625712128",
    "response": "base64",
    "theme": "dark",
    "gradient": "ocean",
    "hideMetrics": true,
    "scale": 2
  }'

Batch Screenshots

POST /screenshot/batch
Auth: Required Credits: N (one per URL, reserved upfront) Content-Type: application/json or multipart/form-data

Generate screenshots for multiple tweets in a single request. Credits are checked and reserved upfront. Max 5 concurrent renders per batch. Shared render options apply to all URLs in the batch.

Batch Size Limits

TierMax URLs per Batch
free10
pro100
business500

JSON Request

{
  "urls": [
    "1617979122625712128",
    "https://x.com/elonmusk/status/1617979122625712128",
    "1234567890"
  ],
  "response": "base64",
  "theme": "dark",
  "gradient": "sunset"
}

CSV Upload (multipart/form-data)

Upload a CSV file with a url column header. Additional render options can be sent as form fields.

curl -X POST https://tweet-shots-api-1084185199991.us-central1.run.app/screenshot/batch \
  -H "X-API-KEY: ts_free_your-key-here" \
  -F "file=@tweets.csv" \
  -F "response=base64" \
  -F "theme=dark"

Tweet Data

GET /tweet/:tweetIdOrUrl
Auth: Required Credits: 1 per request Response: application/json

Retrieve raw tweet data from Twitter's syndication API. Returns the tweet object including text, author info, entities, media, and public metrics. Useful for inspecting tweet data before rendering.

Response

{
  "success": true,
  "tweetId": "1617979122625712128",
  "data": {
    "id_str": "1617979122625712128",
    "full_text": "The tweet text content...",
    "user": {
      "name": "Display Name",
      "screen_name": "username",
      "profile_image_url_https": "https://..."
    },
    "created_at": "2023-01-24T...",
    "entities": { ... },
    "public_metrics": {
      "reply_count": 123,
      "retweet_count": 456,
      "like_count": 7890
    }
  }
}

Example

curl "https://tweet-shots-api-1084185199991.us-central1.run.app/tweet/1617979122625712128" \
  -H "X-API-KEY: ts_free_your-key-here"

Demo Endpoint

GET /demo/screenshot/:tweetIdOrUrl
Auth: None (public) Rate Limit: 5 req/min per IP Response: Binary image (PNG only)

A public endpoint for testing the screenshot API without an API key. Great for evaluating the service before signing up. Responses are cached for 5 minutes.

Supports the same visual parameters as GET /screenshot except custom fonts (fontUrl, fontBoldUrl, fontFamily) and logo (logo) which are excluded to prevent SSRF abuse on the public endpoint.

Supported Parameters

theme, dimension, format, scale, outputWidth, gradient, bgColor, textColor, linkColor, hideMetrics, hideMedia, hideDate, hideVerified, hideShadow, hideQuoteTweet, showUrl, padding, radius, frame, gradientFrom, gradientTo, gradientAngle, thread

Example

curl "https://tweet-shots-api-1084185199991.us-central1.run.app/demo/screenshot/1617979122625712128?theme=light&gradient=sunset" \
  -o demo.png

Parameters Reference

A consolidated reference of all render parameters. These apply to GET query parameters and POST JSON body fields unless otherwise noted.

GET vs POST types: For GET requests, boolean parameters must be the exact strings "true" or "false" (not 0/1/yes/no). For POST JSON bodies, use proper boolean values (true/false) and numbers.

NameTypeDefaultValuesDescription
theme string dark dark, light, dim, black Card color theme
dimension string auto auto, instagramFeed, instagramStory, instagramVertical, tiktok, linkedin, twitter, facebook, youtube Canvas size preset
format string png png, svg Output image format
scale number 2 1, 2, 3 Resolution multiplier
outputWidth number — 50–5000 Final PNG pixel width (overrides scale)
gradient string — sunset, ocean, forest, fire, midnight, sky, candy, peach Background gradient preset
bgColor string — #rrggbb Custom background color
textColor string — #rrggbb Custom text color
linkColor string — #rrggbb Custom link/hashtag color
hideMetrics boolean false true, false Hide like/retweet/reply counts
hideMedia boolean false true, false Hide embedded images/videos
hideDate boolean false true, false Hide tweet timestamp
hideVerified boolean false true, false Hide verification badge
hideShadow boolean false true, false Remove card drop shadow
hideQuoteTweet boolean false true, false Hide quoted tweet
showUrl boolean false true, false Show tweet URL in the card
padding number 20 0–100 Padding around card (px)
radius number 16 0–100 Card border radius (px)
fontFamily string — Any (max 100 chars) Custom font family name
fontUrl URL — Valid URL Custom regular font URL
fontBoldUrl URL — Valid URL Custom bold font URL
logo URL — Valid URL Brand logo image URL
logoPosition string — top-left, top-right, bottom-left, bottom-right Logo placement
logoSize number 40 16–200 Logo size in pixels
frame string — phone Device frame mockup
gradientFrom string — #rrggbb Custom gradient start color
gradientTo string — #rrggbb Custom gradient end color
gradientAngle number — 0–360 Custom gradient angle (degrees)
thread boolean false true, false Render entire tweet thread
response string image image, base64, url Response format (POST only)

Styling Guide

tweet-shots offers extensive visual customization. Here is an overview of the main styling features.

Themes

Four built-in themes control the card's color scheme:

ThemeBackgroundBest for
darkDark blue (#15202b)Default Twitter dark mode look
lightWhite (#ffffff)Light backgrounds, print, documents
dimDark gray (#1e2732)Softer dark mode, less contrast
blackPure black (#000000)OLED displays, maximum contrast

Gradient Backgrounds

Eight preset gradients wrap around the tweet card, adding visual depth. The card itself retains its theme background color.

NameColors
sunsetRed to yellow
oceanPurple to violet
forestTeal to green
fireRed to orange
midnightDark gray to charcoal
skyBlue to light blue
candyPink to gold
peachCream to salmon

Custom Gradients

Use gradientFrom, gradientTo, and gradientAngle together to create your own gradient background. These override any preset gradient.

?gradientFrom=%23ff6b6b&gradientTo=%23feca57&gradientAngle=135

Custom Colors

Override individual colors within the card. These work independently of the selected theme.

  • bgColor — Card background color (overrides theme)
  • textColor — Main text color (overrides theme)
  • linkColor — Hashtag, mention, and URL color (overrides theme)

Dimension Presets

The tweet card always renders at 550px readable width. When a dimension preset is selected, the canvas resizes to the preset dimensions and the card is centered within it.

PresetCanvas SizeUse Case
auto550px (width only)Default, flexible height
instagramFeed1080 x 1080Instagram square post
instagramStory1080 x 1920Instagram / TikTok story
instagramVertical1080 x 1350Instagram portrait post
tiktok1080 x 1920TikTok video thumbnail
linkedin1200 x 627LinkedIn shared image
twitter1200 x 675Twitter card image
facebook1200 x 630Facebook shared image
youtube1280 x 720YouTube thumbnail

Phone Frame

Set frame=phone to wrap the screenshot in a phone device mockup. Works with all themes and gradients.

Custom Fonts

Provide fontUrl and fontBoldUrl pointing to WOFF or TTF font files, along with fontFamily to name the font. The API fetches the fonts at render time.

Custom fonts are not available on the demo endpoint to prevent SSRF abuse.

Branding

Add a logo watermark with the logo parameter (URL to an image). Control placement with logoPosition (e.g., bottom-right) and size with logoSize (default 40px, range 16-200px).

Error Handling

All errors return a consistent JSON format:

{
  "error": "Human-readable description of the problem",
  "code": "SCREAMING_SNAKE_CASE",
  "requestId": "uuid-for-support-correlation"
}

Validation errors include an additional details array with per-field messages:

{
  "error": "Request validation failed. Check the details field for specific issues.",
  "code": "VALIDATION_ERROR",
  "details": [
    { "field": "theme", "message": "Invalid enum value. Expected 'light' | 'dark' | 'dim' | 'black'" }
  ],
  "requestId": "abc-123"
}
